# /meta-reflect

扫描 MISTAKES.md，找错误模式，写入 META_REFLECTION.md，并合并相似条目。

## 执行步骤

1. 读取 MISTAKES.md 全文
2. 统计错误类别分布（思路错误 / 代码错误 / 工具误用 / 需求误解）
3. 找出重复出现的根因（同一底层假设导致多次错误）
4. 生成反思报告，追加到 META_REFLECTION.md：
   - 本周期错误模式表（模式 / 次数 / 典型案例）
   - 根本原因分析
   - 系统性改进措施（具体、可执行）
5. 将相似的 MISTAKES 条目合并（保留一条主条目，其余标注"已合并"）
6. 更新 META_REFLECTION.md 的 `> 最后更新` 日期

## 触发时机

MISTAKES.md 每积累约 20 条时触发（由用户或 hook 提示）。

```markdown
|------|------|---------|
| ... | N | ... |

...

1. ...
```
