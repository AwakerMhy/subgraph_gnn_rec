# /simplify-pass

读取最近 N 个 commit 的 diff，找重复/死代码/过度抽象/未使用的 import，提出简化建议。

## 执行步骤

1. 运行 `git log --oneline -10` 查看最近 10 个 commit
2. 运行 `git diff HEAD~5 HEAD -- src/` 获取最近 5 个 commit 的代码变化
3. 分析：
   - 重复代码（3处以上相似逻辑且尚未抽象）
   - 死代码（定义但未调用的函数/变量）
   - 过度抽象（只调用一次的辅助类/函数）
   - 未使用的 import
   - 过于防御性的 try/except
4. 按优先级排列简化建议，每条说明理由
5. **等待用户逐条确认**后再执行，不自动修改

## 触发时机

每完成 ~5 个任务或一个 Phase 里程碑后运行一次。

## 注意

- 简化建议必须有明确的改动范围（file:line）
- 不要建议"未来可能有用"的抽象
